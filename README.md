# BotRGCN-Het: Adaptive Aggregation for Heterophilic Bot Detection

> A minimal modification to BotRGCN that learns when to ignore noisy neighbors.

Read the full research story in [`WRITEUP.md`](WRITEUP.md) or the one-page summary in [`RESEARCH.md`](RESEARCH.md).

---

## The short version

We show that BotRGCN's main weakness on TwiBot-20 is **not** isolated or low-degree nodes. It is **heterophily**: users are frequently connected to accounts of the opposite class, so mean aggregation corrupts their representations.

We fix this with a learned soft-contrast gate that mixes the neighborhood mean with the ego node:

$$m_r(v) = \mathrm{low}_r(v) + \beta_r(v) \cdot (W_r h_v - \mathrm{low}_r(v))$$

This improves BotRGCN's test F1 macro from **0.8218** to **0.8259** and reduces the dominant human→bot false-positive rate in the most heterophilic bucket.

---

## Why this matters

![Isolated-node hypothesis](results/figures/fig_01_isolated_hypothesis.png)

**Left:** the usual story — isolated nodes are hard. **Right:** the reality on TwiBot-20 — almost no labeled node is isolated; the crawler capped neighbors at ~10 per direction.

![Smoothing problem](results/figures/fig_03_smoothing_problem.png)

Under heterophily, mean aggregation pulls a human node toward its bot neighbors, making it harder to classify.

---

## Main result

| Model | F1 macro | AUC |
|---|---:|---:|
| BotRGCN | 0.8218 ± 0.0025 | 0.9068 ± 0.0025 |
| **SoftContrastBotRGCN-global** | **0.8259 ± 0.0009** | 0.9041 ± 0.0038 |

See [`WRITEUP.md`](WRITEUP.md) for full results, analysis, and figures.

## Reproduction

Requires Python 3.14+ and `uv`.

```bash
uv sync

# Run all analyses
uv run python src/degree_bucket_analysis.py
uv run python src/heterophily_analysis.py
uv run python src/heterophily_fix.py

# Generate figures
uv run python src/writeup_figures.py
```
